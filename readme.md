# NextGuard

누가 필요한가?
- 산업 안전 관리자

왜?
- 산업 안전 관리자가 여러 공장 업체를 돌아다니면서, 산업 안전 법에 위반하는 현장의 모습을 찍고, 왜 부적절한지 그 이유를 작성하고 그와 관련된 법률을 관리자 스스로가 작성해야한다.
- 이런 번거로운 작업을 회피하고 더 정확한 시스템을 구축하기 위해서 AI Agent 시스템을 도입한다.

언제?
- 산업 안전 관리자가 Plant 위험성평가 현장 출장 검사 시에 필요 혹은 현장 사진 추출 후 사무실에서 보고서 작성을 위해 필요

무엇을?
- 산업 안전 처벌법 방지를 위한 AI Agent 시스템

어떻게 개발?
- MultiModal LLM으로 부적절한 산업 재해 사진을 적합하게 변경
- 산업 안전 위반 관련 사진 데이터셋은 AIHUB에 존재하는 것으로 판별 
- RAG 시스템: RDBMS (Postgresql or MariaDB) 또는 GraphDB (Neo4j)를 사용해서 사용자 질의에 대한 처리 고도화 - ex(법 조항 근거 검색)
- MCP 통신을 이용해서 외부 소스 참고 또는 MCP Server + MCP Client에 대한 사용은 추후에 고려 
- 부적합한 사항을 조금 더 고도화해서 작성할 수 있도록 AI Agent 구현 (위험성 평가와 관련된 Workflow를 임의 구현하며, `황한별 선임`에게 피드백 평가 후 기준치 이상 시 통과 그렇지 아니한 경우, WorkFlow 재검토 및 수정)

요구 사항
1. 점검결과 보고서는 HWPX 문서 형태로
2. 현장에서 점검 사진과 부적합 사항 내용을 점검자가 작성하여 업로딩.
3. 업로딩 되면, AI(제미나이 2.5)가 점검 사진과 부적합 사항을 자동 분석.(자동분석은 26년 법제처 중대재해처벌법, 산업안전보건법, 산업안전보건기준 규칙, 고용노동부 고시 범위에서 분석)
4. 자동 분석을 통해 개선 사항 내용과 개선된 사진 및 그에 따른 법령근거(조항 및 세부내용)를 해당되는 작성란에 자동 업로딩.
